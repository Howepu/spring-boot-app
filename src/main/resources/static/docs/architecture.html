Архитектура приложения

Общая архитектура

Spring Boot Ollama Integration построен на основе классической многослойной архитектуры Spring Boot с дополнительными компонентами для интеграции с Ollama API.

Основные слои приложения:

  1. Представление (Presentation Layer) - REST API контроллеры и веб-интерфейс на Bootstrap
  2. Бизнес-логика (Service Layer) - сервисы для обработки запросов и управления данными
  3. Доступ к данным (Data Access Layer) - репозитории для работы с БД
  4. Интеграция (Integration Layer) - клиент для Ollama API

Компоненты системы

1. Контроллеры

  • InsightController - обрабатывает HTTP-запросы для управления инсайтами
  • WebViewController - контроллер для веб-интерфейса

2. Сервисы

  • InsightService - бизнес-логика для работы с инсайтами
    • Создание новых инсайтов
    • Получение результатов анализа
    • Управление статусами запросов
  • NeuralApiService - интеграция с Ollama API
    • Отправка запросов в Ollama API
    • Обработка ответов от нейросети
    • Управление параметрами запросов

3. Репозитории

  • InsightRepository - доступ к данным инсайтов в БД
  • UserRepository - доступ к данным пользователей

4. Конфигурации

  • OllamaApiConfig - настройки подключения к Ollama API
  • SecurityConfig - настройки безопасности приложения
  • WebConfig - настройки веб-слоя
  • MetricsConfig - настройки сбора метрик для мониторинга

Интеграция с Ollama API

Интеграция с Ollama API осуществляется через WebClient из Spring WebFlux, который обеспечивает реактивное взаимодействие с API.

Основные параметры интеграции:

  • ollama.api.url - базовый URL API (по умолчанию http://localhost:11434)
  • ollama.model - используемая модель (по умолчанию llama2:latest)

Пример запроса к Ollama API:

{
    "model": "llama2:latest",
    "prompt": "Анализ продаж за июнь 2025",
    "stream": false,
    "options": {
        "temperature": 0.7,
        "num_predict": 500
    }
}

Обработка результатов:

Результат анализа, полученный от Ollama API, сохраняется в базе данных и связывается с соответствующим запросом инсайта.

Модель данных

Основные сущности:

1. Insight

Сущность для хранения запросов и результатов анализа:

@Entity
@Table(name = "insights")
public class Insight {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(nullable = false)
    private String query;
    
    @Column(columnDefinition = "TEXT")
    private String result;
    
    @Column(name = "created_at")
    private LocalDateTime createdAt;
    
    @Enumerated(EnumType.STRING)
    private InsightStatus status;
    
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "user_id")
    private User user;
}

2. User

Сущность для хранения данных пользователей:

@Entity
@Table(name = "users")
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    
    @Column(nullable = false, unique = true)
    private String username;
    
    @Column(nullable = false)
    private String password;
    
    @ElementCollection(fetch = FetchType.EAGER)
    @CollectionTable(name = "user_roles", joinColumns = @JoinColumn(name = "user_id"))
    @Column(name = "role")
    private Set roles = new HashSet<>();
}

Статусы запросов инсайтов:

public enum InsightStatus {
    PENDING,    // Ожидает обработки
    PROCESSING, // В процессе анализа
    COMPLETED,  // Анализ завершен успешно
    FAILED      // Ошибка при анализе
}

Безопасность

Приложение использует Spring Security для аутентификации и авторизации пользователей.

Основные принципы безопасности:

  • Аутентификация на основе форм входа
  • Шифрование паролей с помощью BCrypt
  • Ролевая модель доступа (ROLE_USER, ROLE_ADMIN)
  • CSRF защита для веб-форм
  • Защита от XSS атак

Настройка безопасности:

@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/", "/css/**", "/js/**", "/docs/**").permitAll()
                .requestMatchers("/api/**").authenticated()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login")
                .defaultSuccessUrl("/dashboard")
                .permitAll()
            )
            .logout(logout -> logout
                .logoutSuccessUrl("/")
                .permitAll()
            );
        
        return http.build();
    }
}
